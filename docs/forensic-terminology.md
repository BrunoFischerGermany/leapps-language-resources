# Terminology and Naming Standards

## Purpose
Defines preferred terminology used across the LEAPPs ecosystem.

Goals:
- Improve consistency
- Reduce duplicate concepts
- Simplify translation
- Standardize artifact terminology
- Provide rationale for naming decisions

## How to use this document
- Contributors should reference this before introducing new labels or headers.
- Existing terms should be reused where possible.
- If a new term is needed, open a discussion/issue first.

## New or Updated Terms
If you find a new term that needs to be consolidated, open an issue to discuss and provide examples. After discussion, the term can be added to this document.

## Timestamp

### Description
A date/time value representing a specific recorded event time.

### Use for
- Date and time values
- Database event times
- File system times
- Message times
- Parsed application timestamps

### Avoid using for
- Durations
- Relative times
- Time zone names
- Human-readable schedules
- Date values without time
- Time values without date

### Preferred forms
- Timestamp
- Timestamps

### Deprecated variants
- time stamp
- TimeStamp
- date/time stamp
